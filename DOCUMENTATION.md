# دليل توثيق مشروع AppointReady (NTG-MED / MedGemma Simulator)

## 📌 1. نظرة عامة على المشروع (Project Overview)
مشروع **AppointReady** هو تطبيق ومحاكي طبي ذكي (Clinical Assistant) يهدف إلى أتمتة عملية جمع بيانات المريض قبل الكشف (Pre-visit Intake). 
يقوم النظام بإجراء مقابلة طبية تفاعلية مع المريض (كتابياً وصوتياً) لجمع الأعراض، التاريخ المرضي، والأدوية الحالية. بناءً على هذه المحادثة، يقوم بإنشاء تقرير طبي شامل بصيغة **PDF**، مما يسهل على الطبيب مراجعة حالة المريض سريعا وتحسين جودة الرعاية الصحية.

تم تعديل هذه النسخة لتعمل بشكل محلي (Local Inference) بالاعتماد على أدوات مفتوحة المصدر، مما يقلل من التكلفة ويزيد من خصوصية بيانات المريض، بدلاً من الاعتماد الكلي على الخدمات السحابية.

---

## 🏗️ 2. معمارية النظام (System Architecture)
يعمل النظام من خلال تكامل عدة مكونات رئيسية:
1. **واجهة المستخدم (Frontend)**: واجهة تفاعلية مبنية باستخدام Streamlit (مفضلة) أو تطبيق React لاستقبال الإدخال الصوتي والنصي من المريض وعرض الردود والتقرير النهائي.
2. **الواجهة الخلفية (Backend)**: خادم Python (Flask/FastAPI) يدير منطق التطبيق وحفظ الجلسات.
3. **وحدة الذكاء الاصطناعي (AI Core)**:
   - **فهم وتوليد النصوص**: عبر نموذج MedGemma المشغل محلياً بواسطة Ollama.
   - **فهم الصوت (STT)**: تحويل كلام المريض إلى نص عبر استدعاء واجهة Groq API المعتمدة على نموذج Whisper.
   - **توليد الصوت (TTS)**: تحويل ردود المساعد الطبي إلى صوت مسموع باستخدام Edge TTS.
4. **توليد التقارير (Reporting)**: معالجة البيانات المجمعة وتحويلها لملف تشخيص مستخدمين مكتبة ReportLab.

## 🛠️ 3. التقنيات المستخدمة (Tech Stack)

| تصنيف التقنية | التقنية / الأداة | الاستخدام في المشروع |
| :--- | :--- | :--- |
| **LLM & Inference** | Ollama | تشغيل نموذج الذكاء الاصطناعي محلياً. |
| **Medical AI Model** | MedGemma (`alibayram/medgemma:4b`) | معالجة النصوص، طرح الأسئلة الطبية، وتلخيص حالة المريض. |
| **Speech-to-Text (STT)** | Groq API + Whisper Large v3 | تحويل الملفات الصوتية للمريض إلى نص موثوق للغة العربية والإنجليزية. |
| **Text-to-Speech (TTS)** | Edge TTS | توليد صوت واقعي لردود المساعد الطبي (يدعم التبديل الآلي للغات). |
| **Backend Framework** | Python 3, Flask, Gunicorn | تقديم الـ APIs وإدارة مسار الجلسات وعمليات الـ Streaming (SSE). |
| **Frontend UI** | Streamlit | بناء واجهة مستخدم المريض التي تدعم الدردشة وتسجيل الصوت وعرض التقرير. |
| **PDF Generation** | ReportLab | إنشاء ملفات PDF منسقة تحتوي على التقرير الطبي وملخص البيانات. |
| **Audio Processing** | Pydub, SoundFile | معالجة الملفات الصوتية واقتطاعها وتهيئتها قبل الإرسال. |
| **Utils & Helpers** | Langdetect, Diskcache, Dotenv | اكتشاف لغة النص (لاختيار صوت TTS)، تخزين مؤقت للنتائج، وحفظ المفاتيح السرية. |
| **Deployment** | Docker | حزم التطبيق داخل حاوية ليسهل تشغيله على أي سيرفر أو بيئة. |

## ⚙️ 4. المتطلبات وإعداد البيئة (Prerequisites & Installation)

لتشغيل المشروع محلياً، ستحتاج إلى إعداد البيئة بالخطوات التالية:

### أ. البرامج المطلوبة
- Python 3.10 أو أحدث.
- [Ollama](https://ollama.com/) مثبت على جهازك.
- مفتاح API مجاني من موقع Groq (لميزة تحويل الصوت إلى نص).

### ب. إعدادات المشروع
1. **تثبيت الاعتماديات المطلوبة**:
   ```bash
   pip install -r requirements.txt
   ```
2. **تحميل نموذج الذكاء الاصطناعي**:
   قم بتشغيل الأمر التالي في التيرمينال ليقوم Ollama بتحميل الموديل الطبي:
   ```bash
   ollama run alibayram/medgemma:4b
   ```
3. **متغيرات البيئة**:
   قم بإنشاء ملف `.env` في المجلد الرئيسي وضع فيه مفتاح الـ Groq الخاص بك:
   ```env
   GROQ_API_KEY=your_groq_api_key_here
   GENERATE_SPEECH=true
   ```

## 🚀 5. تشغيل المشروع (Running the Project)

هناك عدة طرق لتشغيل المشروع بناءً على الواجهة المفضلة لك:

### أ. تشغيل واجهة Streamlit (التجربة الأسهل)
في التيرمينال، اكتب:
```bash
streamlit run str_app.py
```
ستفتح واجهة مستخدم في المتصفح تتيح لك الدردشة الصوتية والكتابية فوراً.

### ب. تشغيل واجهة Flask للـ APIs (للربط مع React)
لتشغيل الباك إند الخاص بالـ APIs والذي يقدم ملفات الـ Static للـ React:
```bash
python app.py
```
أو عبر الـ Script الجاهز:
```bash
bash run_local.sh
```

## 🧩 6. تفاصيل الوحدات الهامة (Core Modules Deep-Dive)

- **`str_app.py`**: هو نقطة الدخول (Entry Point) لتطبيق Streamlit، يحتوي على منطق عرض الرسائل، زر تسجيل الصوت، وإنشاء وتنزيل التقرير الصادر من وحدة التوليد كملف PDF.
- **`app.py`**: ملف الخادم لـ Flask الذي يُعرّف الـ Routes ويتيح اتصال الـ Frontend بالباك إند عبر الـ APIs أو عمليات البث (Streaming).
- **`medgemma.py`**: ملف وسيط للتواصل مع واجهات برمجة تطبيقات Ollama محلياً. يحتوي على إعدادات الموديل وكيفية بناء واستخلاص النصوص والردود.
- **`stt_service.py`**: وحدة مسؤولة عن استدعاء Groq API ومعالجة الملف الصوتي المُرسَل لتحويله إلى نص قابل للفهم. يستعين بـ Prompts خاصة لتشغيل اللغة العربية بشكل أفضل.
- **`edge_ai_tts.py`**: وحدة تقوم بتوليد الصوت الاصطناعي بناءً على رد الذكاء الاصطناعي. مبرمجة بتبديل الصوت (Voice) تلقائياً بناءً على محتوى الجملة (AR/EN).
- **`interview_simulator.py`**: تعتبر محرك الحوار وتتبع حالة كل جلسة (Session). تنظم الأسئلة التي يجب طرحها للمريض وتقرر متى تنتهي المقابلة لبدء توليد تقرير PDF.
